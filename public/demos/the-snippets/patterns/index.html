1. モーダル推奨 中級

背景クリック・Escキーで閉じる、アクセシブルなダイアログ。

使いどころ: 削除の確認、ログイン／登録フォーム、画像拡大など、操作を一時中断して注意を1点に集めたいとき。

デモ

確認

Escキー・背景クリックでも閉じます。

そのまま動く最小実装。 HTML・CSS・JSが揃っているので、3つ貼ればページで動く。data属性で開閉を結線。

<button class="btn" data-open="dlg1">開く</button>

<dialog id="dlg1" class="modal">
  <h3>確認</h3>
  <p>本当に実行しますか？</p>
  <button class="btn" data-close>閉じる</button>
</dialog>
.modal {
  border: none;
  border-radius: 14px;
  padding: 24px;
  max-width: 380px;
}
.modal::backdrop {           /* 背景の暗幕 */
  background: rgba(0, 0, 0, .5);
}
const dlg = document.getElementById('dlg1');

document.querySelector('[data-open="dlg1"]')
  .addEventListener('click', () => dlg.showModal());

dlg.querySelector('[data-close]')
  .addEventListener('click', () => dlg.close());
// Escキーは <dialog> が標準で対応

コード解説

  • HTML/CSS/JS が3点セット。初心者でもこのまま貼れば動くのが狙い。
  • data-open="dlg1" / data-close でトリガーを結線（onclick直書きを避け拡張しやすく）。
  • <dialog> + showModal() で背景暗転・Escキーは標準対応。
  • ::backdrop 擬似要素で暗幕の見た目を制御できる。

実利用向け・汎用化。 ページ内に何個モーダルがあっても1つの仕組みで動く。イベント委譲なので後から追加した要素も自動対応＝拡張しやすい。

    <h3>削除しますか？</h3>
    <p class="muted">この操作は取り消せません。</p>
  </div>
  <menu class="modal__actions">
    <button class="btn ghost" data-close-modal>キャンセル</button>
    <button class="btn" data-close-modal>削除する</button>
  </menu>
</dialog>

  border: none; border-radius: 14px; padding: 0;
  max-width: 420px; width: 90vw;
}
.modal::backdrop { background: rgba(0, 0, 0, .5); }
.modal__body { padding: 24px 26px; }
.modal__actions {
  display: flex; gap: 10px; justify-content: flex-end;
  margin: 0; padding: 16px 26px;
  border-top: 1px solid rgba(0, 0, 0, .08);

// 1か所だけで全モーダルを制御（イベント委譲）
document.addEventListener('click', (e) => {
  const opener = e.target.closest('[data-open-modal]');
  if (opener) {
    document.getElementById(opener.dataset.openModal)?.showModal();
  }
  if (e.target.closest('[data-close-modal]')) {
    e.target.closest('dialog')?.close();
  }
});

// 背景（バックドロップ）クリックで閉じる
document.querySelectorAll('dialog.modal').forEach((dlg) => {
  dlg.addEventListener('click', (e) => {
    const r = dlg.getBoundingClientRect();
    const inside = e.clientX >= r.left && e.clientX <= r.right
                && e.clientY >= r.top  && e.clientY <= r.bottom;
    if (!inside) dlg.close();
  });
});

コード解説

  • イベント委譲で1つのリスナーが全モーダルを担当。動的に追加した要素も結線不要で動く。
  • data-open-modal="ID" / data-close-modal で宣言的に開閉を指定＝HTMLを増やすだけで拡張可。
  • 背景クリック判定は本体の矩形外かどうかで実装（getBoundingClientRect）。
  • BEM風クラスで見た目を分離。複数の確認・フォームモーダルに使い回せる。

本番品質・拡張API。 confirmDialog() が Promise を返すので await で結果分岐できる。フォーカス復帰・スクロールロック・inert・reduced-motion 対応。

// 結果を待てる確認ダイアログ（Promiseを返す）
function confirmDialog(dlg) {
  const last = document.activeElement;            // 復帰用
  document.body.style.overflow = 'hidden';        // 背景スクロール固定
  document.querySelector('main')?.setAttribute('inert', '');
  dlg.showModal();
  dlg.querySelector('button')?.focus();

  return new Promise((resolve) => {
    dlg.addEventListener('close', () => {
      document.body.style.overflow = '';
      document.querySelector('main')?.removeAttribute('inert');
      last?.focus();                              // 元の位置へ戻す
      resolve(dlg.returnValue === 'ok');
    }, { once: true });
  });
}

// 使い方：await で分岐できる＝呼び出し側がシンプル
del.addEventListener('click', async () => {
  const ok = await confirmDialog(document.getElementById('confirm'));
  if (ok) { /* 実際の削除処理 */ }
});

コード解説

  • <form method="dialog"> でボタンの value が returnValue に入る＝結果判定が簡潔。
  • confirmDialog() は Promiseを返すので const ok = await … で分岐。呼び出し側を拡張しやすい。
  • フォーカス復帰・inert・overflow:hidden スクロールロックで本番のアクセシビリティ要件を満たす。
  • prefers-reduced-motion で開閉アニメを抑制＝動きに敏感な人に配慮。

2. パスワード表示切替推奨 中級

アイコンで input の type を password ⇄ text に切り替え。

使いどころ: ログイン・登録・パスワード変更で、入力文字を確認させ打ち間違いを減らしたいとき。

デモ

そのまま動く最小実装。 HTML/CSS/JSの3点。最初から addEventListener と aria-label 更新で保守的に。

<div class="field">
  <input type="password" id="pw" placeholder="パスワード">
  <button type="button" id="pwBtn"
          aria-label="パスワードを表示">表示</button>
</div>
.field { position: relative; max-width: 320px; }
.field input {
  width: 100%;
  padding: 10px 64px 10px 12px;
}
.field button {
  position: absolute; right: 6px; top: 50%;
  transform: translateY(-50%);
  border: none; background: none; cursor: pointer;
}
const pw  = document.getElementById('pw');
const btn = document.getElementById('pwBtn');

btn.addEventListener('click', () => {
  const show = pw.type === 'password';
  pw.type = show ? 'text' : 'password';
  btn.textContent = show ? '隠す' : '表示';
  btn.setAttribute('aria-label',
    show ? 'パスワードを隠す' : 'パスワードを表示');
});

コード解説

  • HTML/CSS/JS が揃った“貼れば動く”最小形。学習用でもこのまま使える。
  • インライン onclick を避け addEventListener で分離＝後で拡張しやすい。
  • 切替のたびに aria-label を更新し、スクリーンリーダーに状態を伝える。
  • CSSはラッパを position:relative、ボタンを absolute で重ねる定石。

  • SVGの <use href> を差し替えて目アイコンを開閉表示。
  • getModifierState('CapsLock') で Caps Lock を検知し警告＝ログイン失敗を予防。
  • autocomplete 指定でパスワードマネージャと正しく連携できる。

function initPasswordToggles(root = document) {
  root.querySelectorAll('[data-password-toggle]').forEach((btn) => {
    const input = btn.closest('.field')
                     .querySelector('[data-password]');

    btn.addEventListener('click', () => {
      const show = input.type === 'password';
      input.type = show ? 'text' : 'password';
      btn.setAttribute('aria-pressed', show);
      btn.querySelector('use')
         ?.setAttribute('href', show ? '#i-eye-off' : '#i-eye');

  • aria-pressed ＋ アイコン差替で状態を視覚・支援技術の両方へ。
  • focus({preventScroll:true}) で切替後もカーソルと表示位置を維持。

コード解説

  • イベントを capture:true でグループに委譲＝項目を後から増減しても結線不要。
  • <details> のネイティブ機能（キーボード/SR）はそのまま維持。

使いどころ: 検索オートサジェスト、リサイズ・スクロール処理など高頻度イベントの負荷を抑えたいとき。

function debounce(fn, delay = 300) {
  let t;
  const wrapped = (...args) => {
    clearTimeout(t);
    t = setTimeout(() => fn(...args), delay);
  };
  wrapped.cancel = () => clearTimeout(t);   // 離脱時に予約解除
  return wrapped;
}

const onSearch = debounce(
  (term) => fetch('/api?q=' + term), 400);
q.addEventListener('input', (e) => onSearch(e.target.value));

コード解説

  • クロージャで t を隠蔽。返り値はどんなイベントにも使える汎用関数。
  • wrapped.cancel() で予約を取り消せる＝画面離脱時の誤発火を防げる。
  • ...args で元の引数をそのまま fn へ転送。
  • 1つ定義すれば検索・リサイズ・保存など各所で使い回せる。

本番品質・拡張。 leading対応 debounce＋AbortControllerで古いリクエストを破棄（検索の競合防止）。

<ul id="results"></ul>
function debounce(fn, wait = 300, { leading = false } = {}) {
  let t, ranLeading = false;
  const wrapped = (...args) => {
    if (leading && !t) { fn(...args); ranLeading = true; }
    clearTimeout(t);
    t = setTimeout(() => {
      t = null;
      if (!ranLeading) fn(...args);
      ranLeading = false;
    }, wait);
  };
  wrapped.cancel = () => { clearTimeout(t); t = null; };
  return wrapped;
}

  ctrl?.abort();
  ctrl = new AbortController();
  const res = await fetch('/api?q=' + term, { signal: ctrl.signal });
  render(await res.json());
}, 400);

コード解説

  • leading オプションで「先頭即時＋末尾1回」を選べる＝用途に応じ拡張可。
  • AbortController で前回 fetch を中断＝遅い応答が新しい結果を上書きする競合を防ぐ。
  • cancel() で離脱時に予約解除＝メモリリーク/誤発火を防止。
  • スクロール追従が要るなら同様の発想で throttle を併用（lodash 採用も実用的）。
